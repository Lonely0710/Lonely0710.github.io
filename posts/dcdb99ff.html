2025.2.9 22:00
参考文章：zsh 安装与配置，使用 oh-my-zsh 美化终端

1.1 说明

此前在 macOS 上的 iTerm2 就一直用的 oh-my-zsh + powerlevel10k，这里记录一下在 linux 下的 root 和 非root 用户下的主题配置。

OS：Debian GNU/Linux 11 (bullseye) x86_64
Shell: zsh 5.8
theme：powerlevel10k

1.2 环境基本配置

1.2.1 安装基本工具

oh-my-zsh 基于 zsh，linux 中一般默认的都是 bash，如果没有则需要提前下载。
设置默认终端为 zsh（注意：不要使用 sudo）。


然后退出服务器重启。

1.2.2 安装 oh-my-zsh

MethodCommand
curlsh -c "$(curl -fsSL https://install.ohmyz.sh/)"
wgetsh -c "$(wget -O- https://install.ohmyz.sh/)"
fetchsh -c "$(fetch -o - https://install.ohmyz.sh/)"
国内curl 镜像sh -c "$(curl -fsSL https://gitee.com/pocmon/ohmyzsh/raw/master/tools/install.sh)"
国内wget 镜像sh -c "$(wget -O- https://gitee.com/pocmon/ohmyzsh/raw/master/tools/install.sh)"

使用 oh-my-zsh 会覆盖现有的 .zshrc.

1.3 配置主题

为了区分 root 和 非root 用户，对于两种设置的主题略有区别。

1.3.1 字体设置

由于在 powerlevel10k 主题中用到了很多 unicode 字符，因此需要设置带有 nerdfonts 的等宽字体。
链接： nerdfonts
宿主机使用的 macOS + iTerm2，字体样式如下：

[!note]

  • Font：0xProto Nerd Font Mono
  • Non-ASCII Font：Hack Nerd Font
    注： 设置 Non-ASCII Font 字体是为了能够让终端中的 icon 显示得大一点，否则在 mono 样式下会显得很小。

1.3.2 配色方案

1.3.3 root 用户

使用 powerlevel10k 主题。


在 ~/.zshrc 设置 ZSH_THEME="powerlevel10k/powerlevel10k"。

1.4 插件安装

只安装了两个插件，也是觉得最实用的，插件太多太臃肿了反而没那么好用，编辑功能大多用 neovim 来解决，所以我实际需要的也就是高亮提示+自动补全。
基本流程都是：

  • 将插件下载到 ~/.oh-my-zsh/custom/plugins ；
  • 然后在 .zshrc 的 plugins 列表中插入。

    1.4.1 zsh -autosuggestions

    zsh-autosuggestions 是一个命令提示插件，当你输入命令时，会自动推测你可能需要输入的命令，按下右键可以快速采用建议。

1.4.2 zsh-syntax-highlighting

zsh-syntax-highlighting 是一个命令语法校验插件，在输入命令的过程中，若指令不合法，则指令显示为红色，若指令合法就会显示为绿色。
